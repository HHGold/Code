使用 ECC (Everything Claude Code) 的開發流程是非常結構化且高效的。以下我用一個 「英文單字測驗系統 (English Quiz App)」 作為範例專案，說明從建立到開發過程中該如何運用這些斜線指令：

第一階段：專案初始化 (Setup)
在您開始寫程式碼之前，必須確保專案目錄已具備 ECC 的「代理本能」。

建立專案資料夾：
powershell
mkdir C:\AI\Code\EnglishQuiz
連結 ECC 能力： 執行您剛剛同步好的腳本：

C:\AI\everything-claude-code\install_ecc.ps1
，並選擇 C:\AI\Code\EnglishQuiz。 這會在專案下建立 .agent/rules，讓我在這個專案中自動啟動專屬規則。
第二階段：規劃與架構設計 (Planning)
指令：/plan

何時使用：當您有一個大想法，但還沒開始動工時。
範例對話：
「/plan 我想建立一個 Python 版的英文單字測驗工具，使用者可以從 JSON 讀取題庫，測驗後顯示分數。請遵循 Clean Architecture。」

指令作用：我會讀取 

plan.md
 的 SOP，為您分析需求、產出系統目錄結構架構圖，並將任務拆解成細小的開發清單。
第三階段：測試驅動開發 (Implementation)
指令：/tdd

何時使用：開始實作具體功能時（例如實作「計算分數」的邏輯）。
範例對話：
「/tdd 實作 QuizEvaluator 類別，根據使用者的答案計算得分，答對 1 題給 10 分。」

指令作用：
RED：先寫一個會失敗的測試案例。
GREEN：撰寫剛好能通過測試的最小程式碼。
REFACTOR：優化程式碼結構，確保簡潔 (DRY)。
第四階段：程式碼稽核與優化 (Review)
指令：/code-review

何時使用：功能完成後，想要確保程式品質。
範例對話：
「/code-review 檢查剛剛寫好的評分系統，是否有符合 Clean Code 準則或效能瓶頸？」

指令作用：我會像資深開發者一樣稽核您的程式碼，檢查命名、耦合度以及是否有潛在 Bug。
第五階段：全面驗證 (Verification)
指令：/verify

何時使用：要確認專案目前的整體健康狀態，或是在提交程式碼前。
範例對話：
「/verify」

指令作用：自動執行所有測試、檢查 Build 是否通過、確認技術文件 (README) 是否有更新，確保專案處於可發布狀態。
第六階段：建立存檔點 (Checkpoint)
指令：/checkpoint

何時使用：完成一個階段性任務，準備進入下一個功能前。
範例對話：
「/checkpoint 完成了核心評分邏輯。」

指令作用：我會總結目前的所有變更，並協助您進行 git commit 或記錄當前進度，方便日後追蹤。
總結指令清單 (ECC 核心指令)
階段	指令	目的
開端	/plan	需求分析與架構設計
開發	/tdd	透過「測試-實作-重構」確保邏輯正確
檢查	/code-review	程式碼品質與安全性稽核
驗證	/verify	自動化測試與環境檢查
存檔	/checkpoint	記錄進度與工作總結
如果您現在想嘗試，我們可以針對您目前開啟的 TravelMonitor 專案執行 /verify 來看看這個專案的現狀！
